# Как править без боли (инструкция для "не программиста")

## 0) Что НЕ надо трогать
- Не трогай файлы в `backend/`, если ты просто хочешь поменять цены/тексты.
- Не трогай `backend/v5/data.py` — он просто читает DATA из JSON.

## 1) Где живёт "настройка" (цены/пакеты/баллы)
Открой файл:
- `frontend/data/data.json`

Это как "таблица в экселе", только в JSON.

### 1.1. Поменять цену пакета
В DATA есть:
- `segments` → "Розница" / "Опт" / "Производитель/Импортёр" → список пакетов.

Ищешь нужный пакет и меняешь `price`.

Пример:
```js
{"name": "Старт", "price": 19800, ...}
```

### 1.2. Поменять "рублей за 1 балл"
Найди:
- `"rub_per_point": 4950`
Поменяй число.

### 1.3. Поменять сколько баллов даёт галочка
Найди:
- `"points_model": { ... }`

Там такие поля как:
- `aggregation_points`
- `org_points_per_extra`
- `big_volume_points`

Меняешь числа.

## 2) Где менять варианты выбора (списки)
Открой файл:
- `frontend/assets/catalogs.js`

Там есть списки:
- `KKT_CATALOG` — производители и модели касс
- `DEVICE_CATALOG` — типы устройств
- `CZ_GROUPS` — товарные группы ЧЗ (для производителя)

Хочешь добавить модель кассы:
- находишь нужного производителя в `KKT_CATALOG`
- дописываешь строку в массив

Пример:
```js
"АТОЛ": ["11Ф", "20Ф", "92Ф", "НОВАЯ_МОДЕЛЬ"]
```

## 3) Как понять, что всё работает
1) Запусти `start.bat`
2) Открой `http://127.0.0.1:8000/`
3) Поставь пару галочек — справа должна меняться сумма

Если что-то сломалось:
- жми F12 → Console → пришли ошибку (одной строкой)

## 4) Если хочешь добавить новую галочку (чуть сложнее)
Нужно 3 места:
1) **State** — добавить поле в объект `state`.
2) **UI** — где рисуются опции:
   - `frontend/assets/ui/checklist.js`
3) **Расчёт** — добавить правило:
   - локально: `frontend/assets/calc.js`
   - и в бэке: `backend/v5/calc.py`

Почему нужно и там и там:
- UI показывает результат сразу
- backend нужен для "истины" и для будущей интеграции/экспорта.

Если ты не хочешь разбираться — просто напиши, что за галочка и сколько она должна добавлять.
